Introdução

Todo código deverá ser avaliado por um revisor de código. Neste caso o papel do revisor é avaliar se o desenvolvedor produziu código que poderá atrapalhar a semântica do projeto ou a manutenção num futuro próximo, também deve verificar se o código produzido é eficiente e atual com relação aos procedimentos utilizados no Laboratório. Boa parte das técnicas empregadas na revisão de código derivam das técnicas desenvolvidas no mozilla.org, veja:

http://www.mozilla.org/hacking/reviewers.html
http://www.mozilla.org/projects/seamonkey/rules/code_review.html

Também cabe ao revisor ensinar, se for o caso, o desenvolvedor novas técnicas e/ou mostrar novas soluções. Uma discussão sobre eventuais mudanças no código sempre deverá acontecer entre o revisor e o desenvolvedor para que este tome conhecimento de o quê e o porquê seu código sofreu mudanças, difundindo assim o conhecimento entre os desenvolvedores.
